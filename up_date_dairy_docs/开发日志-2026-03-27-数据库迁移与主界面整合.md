## 三、闪卡主界面今天完成了哪些整合

### 1. 复习页正式切成 API 直连
今天主闪卡页已经改成：

- 页面直接请求 `/api/cards/due`
- 评分后直接写 `/api/cards/<id>/review`
- 今日回顾直接请求 `/api/review/today`

这意味着复习结果现在是实时落库，不再依赖旧的整库同步思路。

### 2. “添加卡片”整合回主界面
之前新增卡片页面是一个独立旧入口，甚至还残留着 `127.0.0.1:8001/upload.html` 这种历史路径。

今天已经完成整合：

- 在主闪卡页顶部增加了“添加卡片”入口
- 点击后会从右侧滑出加卡面板
- 面板里承载原先的加卡页面逻辑
- 文本建卡和图片建卡都仍然可用
- 不再需要用户单独找旧页面入口

同时补上了快捷键：

- `A` 打开添加卡片面板
- 再按一次 `A` 关闭面板
- `Esc` 也可关闭

### 3. 新增卡片成功后能回流主页面
今天还补了一个很关键的小体验：

- 在右侧加卡面板里创建成功后
- 主页面会收到消息
- 新卡会直接并入当前系统状态

这使得新增卡片不再像“跳出去做另一件事”，而是成为主界面流程的一部分。

## 四、今天修掉的重要问题

### 1. Flask 环境问题
一开始直接用：

```powershell
py app.py
```

会报：

- `ModuleNotFoundError: No module named 'flask'`

最终确认原因是系统默认 Python 环境没有安装项目依赖，而当前可用环境是：

```powershell
C:\Users\surface\miniconda3\python.exe app.py
```

### 2. 主页面能打开但脚本不加载
曾出现主页面能打开、但卡片不加载的问题。

根因是：

- `/script.js` 之前没有正确被 Flask 服务
- `fsrs.js` 模块资源路径也没有正确暴露

后来在 `app.py` 中补了静态资源路由，问题解决。

### 3. AI 建卡接口报 `KeyError`
今天一个很关键的问题是 AI 接口报：

- `KeyError: '\n  "expression"'`

最终定位到原因：

- prompt 模板使用了 `.format(...)`
- 模板里本身又写了 JSON 示例
- Python 把 JSON 花括号当成了格式占位符

修复方式：

- 不再直接用 `.format(...)`
- 改成只替换我们自己定义的两个占位字段

这一步修复后：

- `/create_from_text` 恢复可用
- `/upload_image` 这条链路也一起受益

### 4. 今日回顾和例句功能退化
数据库迁移之后，一开始为了保住主流程，今日回顾被临时简化了，导致这些能力退化：

- 例句字段读取不完整
- 今日回顾详情展示不完整
- 焦点导航缺失
- 分步 reveal 缺失
- 全局扫描快捷键缺失
- review 模式里例句的 reveal 节奏丢失

今天已经把这些旧版关键交互补回来了。

## 五、今日回顾交互恢复了哪些旧能力

今天“今日回顾”已经恢复到接近旧版机制的状态，主要包括：

- `↑ / ↓` 切换焦点卡片
- 鼠标滚轮切换焦点
- 焦点移动时列表自动滚动居中
- `→` 分四步揭示
  - 1：definition
  - 2：context
  - 3：translation
  - 4：打开例句
- `←` 往回收一层
- `,` 全局显示 definition
- `.` 全局显示 context
- `/` 全局显示 translation
- 例句弹窗中：
  - `→` 切换翻译显示
  - `←` / `Esc` / `Q` 关闭
- review 模式下打开例句时，默认先不显示中文

这部分工作非常值，因为它直接影响学习效率与操作流畅度。
