# 开发日志（2026-03-23）

## 背景与问题
- 现阶段高亮定位依赖 DOM path + offset：一旦对正文做“编辑/删除/重新排版”，高亮锚点可能失效或偏移（属于 MVP 阶段的已知限制）。

## 设想：让“生成文章”可逆（回到可编辑导入页）
目标：即使文章已经结构化/排版/清理并保存，也能随时回到“导入编辑页”（可继续粘贴/清洗/预览/重新生成），从而把“不可逆的排版结果”改为“可回溯的派生结果”。

### 核心思路
- 把文章分成两层数据：
  - `content_raw`：源内容（用户粘贴的原始 HTML/纯文本快照，尽量原样保留）
  - `content_html`：派生内容（清洗 + 结构化排版 + 手工编辑/删除后的最终阅读稿）
- 阅读器侧新增“回到导入/源编辑”入口：
  - 直接打开现有导入弹窗
  - 自动填充标题/来源 URL/原始粘贴内容（`content_raw`）
  - 预览区按当前清洗规则重新生成结构化内容（并允许再次手动选“正文从此开始”）

### 版本与高亮的处理建议（两种策略）
1) **重新生成即新版本（推荐）**
   - 重新生成会产生新的 `content_html` 版本号（数据库已有 `version` 字段可用）
   - 高亮与版本绑定：切到新版本时提示“高亮需要重建/迁移”
   - 优点：避免“改稿后高亮错位”的静默错误
2) **重新生成覆盖旧稿（更简单，但风险更大）**
   - 直接覆盖 `content_html`
   - 保留高亮但提示可能错位（当前就是这种风险）

> 结论：用“源内容可回溯 + 重新生成新版本”可以把高亮不稳定的问题，从“不可控”变为“可解释、可回退”。

## 今日完成（已落地）

### 1) 导入清洗与结构化排版（B + C；方案 1 + 2）
- 支持从网页粘贴 **HTML / 纯文本**，解析为 blocks 后再结构化组装为：标题 / deck（摘要）/ 作者 / 时间 / 正文。
- 增加手动控制“正文从此开始”，解决标题/正文分离不稳定的问题。
- 修复“纯文字粘贴被合并成一大段”的根因：HTML 文本节点内换行不再被压扁，能更稳定地分段。
- 增强去噪：过滤 Newsletter / Logo / À lire aussi / Vidéo 等干扰项；过滤 URL 行；加入 Le Monde 分享/版权声明类文本的去噪规则（勾选去噪时生效）。

### 2) 阅读器经典层级（C1）与标题观感提升
- 调整阅读区宽度与层级对比：标题字距/行高、deck/meta 颜色分层、断行策略（`hyphens:auto`）。
- 自动为法语文章设置 `lang="fr"`（断行与排版更自然）。

### 3) 阅读器内“局部清理”（替代“一键清理/重新清洗”）
- 编辑/清理模式：悬停 `×` 删除图片/模块；支持撤销（按钮 + `Ctrl/Cmd+Z`）。
- 覆盖更多媒体类型（`figure/picture/img/video/iframe/embed/object`）与顶层模块删除。
- “重新清洗”产品入口移除（避免误用）。

### 4) 方案 E1：双层编辑（段落级编辑）
- 编辑模式下提供 `✎`：点击仅让该段落进入 `contenteditable`，避免整篇富文本编辑造成结构混乱。
- 支持编辑正文段落；并支持编辑标题 `h1` 与 deck 段落（与正文一致）。
- 保存时剥离编辑/清理用按钮与属性，写回 `content_html`。

### 5) 后端支持
- 增加 `PATCH /api/articles/<id>`：用于保存清理/编辑后的 `content_html`（及标题等字段）。
