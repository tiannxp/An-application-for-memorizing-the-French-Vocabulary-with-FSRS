# Reader MVP：导入排版（B+C）验收清单

记录日期：2026-03-22（Europe/Paris）

本清单用于验证 Reader MVP 的“网页粘贴导入”排版质量，覆盖：

- C：导入区使用 `contenteditable`，粘贴时优先捕获 HTML
- B：HTML 清洗与归一化时保留块级边界（段落/标题/列表/引用）

## 入口

- 启动后打开：`/reader`
- 导入弹窗：点击右上角“导入”

## 核心验收点

### 1) 粘贴捕获

1. 从网页正文区域复制一段包含多段落的文章
2. 粘贴到导入弹窗的正文区域
3. 观察“导入模式”下方提示：
   - 应显示捕获到 `HTML`（若网页提供）
4. 点击“预览”：
   - 应能看到清洗后的排版（段落已拆分，而非一整坨）

### 2) 段落/标题结构

- 标题/小标题应呈现为 `h2/h3`（或至少成为独立块）
- 正文每一自然段为一个 `<p>`
- 连续 `<br>` 不应把整篇压扁为一个段落

### 3) 去噪（推荐打开）

勾选“去噪（推荐）”后，应删除或显著减少以下内容块：

- Newsletter/订阅提示
- “À lire aussi”等推荐阅读入口
- “Vidéo…”等视频入口
- “Logo …”等站点装饰文本
- 重复出现的摄影说明/图片说明（连续重复应去重）

### 4) 回退与对照

- 切换“编辑/预览”不会丢失粘贴内容
- 切换“自动/HTML/纯文本”可改变清洗策略

### 5) 高亮稳定性（回归）

1. 保存导入并打开文章
2. 选中文本创建高亮
3. 刷新页面/重新打开文章
4. 高亮应可复现（同一快照下）

## 推荐样例（手工）

使用任一新闻站/博客文章即可；重点是包含：

- 多段落
- 若干“噪声块”（栏目、订阅、推荐阅读、图片说明）

如遇仍“段落粘连”，优先记录：

- 你粘贴的是网页正文还是整页
- 浏览器是否提供 `text/html`（提示里是否捕获到 HTML）
- 预览里是哪类结构被压扁（div 套娃 / br 连续 / 行内 span 极多）
