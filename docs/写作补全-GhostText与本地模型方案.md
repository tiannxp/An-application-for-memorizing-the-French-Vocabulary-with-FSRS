# 写作补全（Ghost Text）：设计记录与实现路径

记录日期：2026-03-21（Europe/Paris）

## 目标

在写作练习页面提供类似 Cursor Tab 的“补全建议”：

- 用户停顿后自动给出一句话级别的续写建议
- 以 ghost text（半透明灰字）显示在光标后
- 按 `Tab` 接受并写入；继续输入则忽略
- 支持注入“最近在学的词”（来自词库/FSRS），让补全自然复现目标词汇

## 技术原理（分层）

### 1) 触发时机检测

- 监听输入事件
- debounce：通常 300-600ms 无输入后触发一次补全
- 取光标前的文本作为 context（可加上部分光标后的文本作为补全约束）

### 2) 补全请求

请求由后端转发到模型（云端或本地），并带上提示词与上下文，例如：

- System: 你是法语写作助手，补全长度控制在一句话以内
- Context: 用户已写内容（光标附近窗口）
- Optional: 最近学习词表，鼓励自然使用

### 3) Ghost Text 渲染

Ghost text 只在 UI 层显示，不写入 textarea/editor 的 value：

- 用一个 overlay 层渲染“用户已有文本 + ghost 文本”
- ghost 文本用半透明颜色
- Tab 接受时才把 ghost 追加到真实文本

关键点：

- 需要跟随光标位置（或至少在末尾场景正确对齐）
- 输入继续发生时要立刻清掉当前 ghost

## 方案对比

### 方案 A：直接调用 Gemini（现有栈复用）

优点：

- 依赖最少，prompt 可控
- 便于注入词库上下文

缺点：

- 延迟 1-3s 会被感知（视网络与模型而定）

适合：

- 做最小可用版本验证体验与 prompt

### 方案 B：Streaming（SSE/分块输出）降低“延迟感”

思路：

- 后端以流式方式把 token/片段推到前端
- 前端边收到边更新 ghost text

价值：

- 即使总耗时不变，体验会明显更接近 Cursor Tab

### 方案 C：本地小模型（Ollama）做低延迟补全

思路：

- 在本机启动 Ollama 服务，后端改为请求 `http://localhost:11434`
- 小模型（1-3B）更有机会做到“接近实时”的补全体验

注意：

- 设备性能与硬件加速能力会显著影响体验
- 下面关于 Surface Pro/SoC/加速的判断属于“经验推断”，需以实际跑分与实测为准（不要把它当成硬指标）

## 针对本项目的增强点：词库注入

因为项目已有 FSRS 词库，可让补全“更像老师”而不是随便续写：

- 从“最近复习/最近新增/易错卡”抽取一组候选词
- 在 prompt 中要求：在合适时自然使用其中 0-2 个
- 限制补全长度：一句话或最多 N 个 token
- 明确：不要胡编事实，不要改变人称/时态风格

示例（伪代码）：

```text
你是法语写作助手。
补全要求：
1) 只补全一句话，保持用户语气与时态。
2) 如果合适，优先自然使用这些词（最多 2 个）：<vocab_list>
3) 不要输出解释，只输出要补全的文本。
```

## 实施路径（推荐）

### 第 1 步：textarea + 非流式（快速验证）

目标：

- 把完整链路跑通（debounce -> 请求 -> ghost text -> Tab 接受）

交付：

- 一个新写作页面（或现有页面新增写作区）
- 一个后端 `POST /api/complete`（先走 Gemini）

### 第 2 步：加入 Streaming（提升体验）

目标：

- 前端边收边渲染 ghost text
- 支持中断：用户继续输入就取消上一条补全

交付：

- 后端改为 SSE（或 fetch ReadableStream）
- 前端做 abort controller + 增量渲染

### 第 3 步：接入本地模型（可选）

目标：

- 抽象“模型适配层”：Gemini 与 Ollama 可切换

交付：

- `.env` 中新增 `LLM_PROVIDER=gemini|ollama`
- `OLLAMA_MODEL=qwen2.5:3b` 等

## Ollama 集成草案

后端请求（思路示例）：

```python
import requests

resp = requests.post(
  "http://localhost:11434/api/generate",
  json={
    "model": "qwen2.5:3b",
    "prompt": prompt,
    "stream": True
  },
  stream=True,
  timeout=60,
)
```

前端：

- 用 `fetch()` + stream reader 或 SSE 客户端逐段读
- 将累计内容更新到 ghost 层

## 风险与注意事项

- 体验一致性：输入继续发生时必须立刻取消/清空补全，否则会“抢输入法”
- 上下文窗口：只发“光标附近”而不是整篇文章，减少延迟与幻觉
- 词库注入要克制：强行塞词会破坏自然表达，应让模型“可用则用”
- 本地模型质量：法语补全质量在小模型上差异很大，需要实测选择

## 待决策点

- 写作区采用什么编辑器：先 `textarea`，还是直接上 CodeMirror/Tiptap
- 只做“句子续写”，还是也做“改写/纠错/更地道表达”
- 词库注入来源：最近复习、低掌握度、指定主题词袋
